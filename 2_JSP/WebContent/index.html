* JSP (Java Server Page)

JSP란, HTML 내에서 자바 코드를 쓸수있는 "자바 언어"로,
Servlet에서의 응답화면 구현을 보다 간단하게 해결가능
=> 즉, JSP의 가장 큰 장점은 Servlet에서는 요청처리(business logic)에만 집중하고
응답화면 만들기(presentation logic)는 JSP에서 집중하게끔 분리가능하다는 것


* JSP Element 표현법

1. JSP 스크립팅 원소(Scripting Element)

JSP페이지 내에서 자바코드를 기술하게 하는 기능

  1. 선언문 : <%! 자바코드 %>
    멤버 변수와 메소드를 선언하기 위해 사용 => 잘 안쓰임
  2. 스크립틀릿 : <% 자바코드 %>
    가장 기본적인 표현법
  3. 표현식(출력식) : <%= 자바코드%>
    자바에서의 값(변수 또는 메소드 호출)을 화면상에 출력하기 위해 사용

2. 지시어(Directive)
<%@ page | include | taglib 속성="속성값" %>

JSP 페이지 전체에 영향을 미치는 정보들 기술할때 쓰임

  1. page 지시어 : 현재 jsp페이지를 처리하는데 필요한 속성들을 기술
    • language : 사용할 언어 유형 지정
    • contentType : 해당 페이지의 형식, 문자셋 지정(html)
    • pageEncoding : jsp페이지 내의 자바코드에 대한 인코딩 방식 지정
    • import : 자바의 import와 같은의미
    • errorPage : 해당 jsp상에 오류 발생시 보여줄 에러페이지 경로지정
    View detail »
  2. include 지시어 : 현재 jsp상에 또다른 jsp를 포함시키고자 할때 사용하는 지시어
    View detail »
  3. taglib 지시어 : JSP 기능을 좀더 확장시킬수 있는 추가적인 라이브러리
