Get 방식으로 요청 후 응답페이지 보기

특징1. Get 방식으로 요청시 URL의 Header 영역에 데이터를 포함시켜서 요청
=> 사용자가 입력한 값들이 URL에 적나라하게 노출
=> 보안유지가 불가능
=> 즉, 로그인이나 회원가입같은경우 get방식 부적합

특징2. Header 영역은 전송하는 길이에 제한이 있음
=> 방대한 데이터를 담았을 경우 초과된 데이터는 절단됨ㄷㄷ
=> 즉, 게시판 작성같은 경우 get방식 부적합

특징3. 장점! url에 데이터가 노출되어있어 즐찾, 북마크 가능.. 그리고 빠름!
=> 즉, 검색 기능은 get방식에 적합
